# Свод Сводов — сборщик записки из данных

Назначение: каждый месяц не заполнять записку руками, а **считать её из
таблицы-источника** и подставлять в шаблон.

## Источники данных (Google Drive)
- **СводСводов2021** (`1lAvQQlw4w0VO2CVySozxOXJ8h73SWbhIGLy1u2mGK1o`) —
  помесячные «Сводные отчёты по ломбарду» по подразделениям (71 колонка).
  Главный источник по ломбарду.
- **Шаблон для Свод Сводов** (`1ElHcISs0dVnlzfnjQcoAsdhDkSMxKdPT48RnrLmLeHU`) —
  шаблон записки.
- **Свод Сводов за Май/Апрель/Март 2026** — готовые результаты (эталоны).
- Авто-ломбард: «АВТОЛМБ НОВАЯ МОТИВАЦИЯ», «Выполнение планов …».

## Инструменты
- `tools/read_svod_xlsx.py` — чтение реальной выгрузки `.xlsx` из 1С;
  восстанавливает многоуровневую шапку в плоские ключи
  («Движение ТМЦ/Конечный остаток» и т.п.). **Основной вход для месячного прогона.**
- `tools/build_report.py` — сборка раздела «Ломбард» из `.xlsx`: вал, актив,
  Δ актива, выдано, перемещено, клиенты, топ-10 прирост/падение → `.md` + `.docx`.
- `tools/parse_svod.py` — парсер «Сводного отчёта» в текстовом виде (md/pipe).
- `tools/verify_may.py` — проверка движка против эталона за Май.
- `tools/md_to_docx.py` — рендер записки в `.docx` (жирный + цвет по знаку, таблицы).
- `tools/make_sample_xlsx.py` — тестовый `.xlsx` из доступных данных (для проверки ридера).

Зависимости: `pip install -r requirements.txt` (python-docx, openpyxl).

## Запуск (месячный прогон)
```
python tools/build_report.py "Сводный отчёт по ломбарду за Май 2026.xlsx"
```
На вход — месячная выгрузка `.xlsx` со всеми подразделениями. Δ за месяц
считается внутри листа (Конечный − Начальный), отдельный файл за прошлый месяц
не нужен; для годового топ-10 — такая же выгрузка за тот же месяц прошлого года.

## Ключевые правила расчёта
- **Δ актива за месяц** = «Движение ТМЦ/Конечный остаток» −
  «Движение ТМЦ/Начальный остаток». Колонку «Изменение» из листа НЕ использовать —
  там местами `#ERROR!`.
- Δ в граммах = то же по «Движение ТМЦ ВМ 585, гр.».
- Цвет в документе: значение со знаком `+` — зелёный, `-` — красный.

## Проверенный результат
На реальных данных майского листа Δ актива, посчитанный движком, совпал с
эталонной запиской **до рубля** для 002-Спортивная, 007-Российская,
018-Цюрупа, 019-Чишмы (4/5 проверенных; 028-Давлеканово — расхождение 10 000,
вероятно ручная правка/перемещение).

## Известное ограничение по доступу
Drive-интеграция (`read_file_content`) отдаёт только верх каждого большого листа
(~35 строк из ~150) — часть подразделений обрезается; книга на 7 МБ целиком не
выгружается. Для полного прогона нужен полный доступ к данным:
1. файл-выгрузка месячного «Сводного отчёта» (`.xlsx`, все подразделения), либо
2. Google Sheets API к книге-источнику (чтение листа целиком).
